# 长图切割与OCR文字提取

## 概述

本技能用于处理长截图（如微信群聊记录、网页长图等），完成以下工作：
1. 将长图按固定高度切割成多张小图
2. 使用视觉模型逐段读取图片内容（比传统OCR对深色主题、低分辨率图片效果更好）
3. 清理乱码，按对话/内容结构整理成可读文本

## 工作流程（SOP）

### 第一步：检测图片信息

```python
from PIL import Image
img = Image.open(input_path)
print(f"图片尺寸: {img.size[0]}x{img.size[1]}")
print(f"图片模式: {img.mode}")
```

- 如果图片高度 > 3000px，属于长图，需要切割
- 如果图片高度 ≤ 3000px，可以直接用视觉模型读取

### 第二步：切割长图

运行切割脚本：

```bash
python3 scripts/split_image.py <input_image> [--output-dir <dir>] [--chunk-height <pixels>]
```

参数说明：
- `input_image`: 输入图片路径（必须）
- `--output-dir`: 输出目录（默认为 `/workspace/split_images`）
- `--chunk-height`: 每个切片的高度（默认 2000px）

脚本会：
- 将长图按指定高度切割
- 自动保存为 `chunk_01.png`, `chunk_02.png`, ...
- 输出切割数量和每个切片的尺寸信息

### 第三步：视觉模型逐段读取（核心步骤）

**关键原则：使用 Read 工具的视觉能力逐段读取图片，而不是用 pytesseract 等 OCR 引擎。**

对于深色主题截图、聊天记录、低分辨率图片，传统 OCR（如 pytesseract）会产生大量乱码。
视觉模型（通过 Read 工具读取图片）的识别准确率远高于传统 OCR。

对每个切片执行：

```
Read tool -> file_path: chunk_XX.png -> target: "提取提示词"
```

**提取提示词模板**（根据图片类型选择）：

#### 微信群聊截图：
```
这是一张微信群聊截图，深色主题。请逐条提取所有聊天消息，格式为：
发言人：消息内容。忽略头像、表情包等图片内容，只提取文字。
如果有时间戳也一并提取。
```

#### 网页截图/文档截图：
```
这是一张网页/文档截图。请提取所有可见的文字内容，保持原有的层级结构和格式。
如果有标题、列表、表格等结构元素，请用Markdown格式保留。
```

#### 通用截图：
```
请提取这张图片中的所有文字内容，保持原有的结构和格式。
```

**并行读取优化**：每次可以同时读取 3 张切片（受系统限制），减少等待时间。

### 第四步：整理与清理

将所有切片的提取结果合并，进行以下清理：

1. **去重**：相邻切片边缘可能有重复内容，需去除
2. **按时间线排序**：如果有时间戳，按时间顺序整理
3. **识别发言人**：标注每条消息的发送者
4. **标注系统消息**：入群提示、撤回消息等单独标注
5. **保留引用/回复关系**：用 `> 引用` 格式标注
6. **保留链接**：URL、文件分享等完整保留
7. **标注图片消息**：用 `[图片 - 描述]` 格式标注（不提取图片内容）

### 第五步：输出最终文件

将整理后的内容保存为 Markdown 文件，结构如下：

```markdown
# 标题（根据内容自动生成）

> 来源说明
> 参与成员列表

---

## 时间段1 — 话题摘要

**发言人：** 消息内容

*时间戳*

---

## 时间段2 — 话题摘要

...

---

## 📎 相关资源汇总

| 资源 | 链接 |
|------|------|
```

输出文件保存到 `/workspace/` 目录下。

## 注意事项

1. **切割高度建议**：
   - 微信聊天截图：1500-2000px（消息密度适中）
   - 网页截图：2000-2500px
   - 文档截图：1500-2000px

2. **视觉模型读取限制**：
   - 每次最多并行读取 3 张图片
   - 如果图片内容模糊，可在 target 提示中强调"请仔细辨认"

3. **乱码清理规则**：
   - 常见OCR噪声：`Biography`、`avd stl) on a)`、随机字母组合
   - 单个无意义字符行直接删除
   - 纯乱码行（无中英文有意义内容）直接删除

4. **文件放置规则**：
   - 切割后的图片 → `/workspace/split_images/`
   - 最终整理文本 → `/workspace/`
   - 中间脚本 → `/data/user/work/`（不保存到workspace）
